Mysql获取随机数据的问题，很纳闷

记得以前找到过方法，而且可用，忘记了，重新google，发现网上提供的方法，有些问题：

一个简单的实现如下：

SELECT FROM table
WHERE id >= (SELECT floor( RAND() ((SELECT MAX(id) FROM table)-(SELECT MIN(id) FROM table)) + (SELECT MIN(id) FROM table)))
ORDER BY id LIMIT 1;
简化测试可以写为：select from table where id>=(select rand()(select max(id) from table)) limit 10

据我测试mysql5.1 如果table有20000条数据，基本没有可能获取到一条大于1000的数据，我也搞不清楚原因，难道是我的mysql版本问题？真是奇怪了。继续测试。
将limit 1 改为10后，结果发现，返回的数据依然不能超过1000，而且结果的id不连续，则说明mysql是逐条执行了where的rand，即执行了20000次，这是我的估计，毕竟我不是mysql专家，不知道如何研究。即便是这样，那么每条的几率依然应该是1/20000分之一的，为何结果却出乎意料？
换了一个mysql服务器，5.1.41，结果依然如此。

找到原来的写法了，很简单：order by rand() limit 10,,如果数据大可以用id in (select rand()max(id) from t,..,..) ，呵呵，这是从别人那看到的。另外附加sql server的写法：select top 10 from tablename order by newid()

2010-06-18

面对改变

最近的一连串加薪、大力度的房产调控、即将实行的药价反暴利等等一系列现象。也在深切地影响着股市，被一些玩弄是非的人那么一鼓捣，让人真的找不到北了，最近和老妈闲聊村里一个以前的百万富翁破产的事情，让我有点茅塞顿开。原来世界在改变，而且国家也真真切的一直在告诉我们，“参业结构要调整”，只不过没有翻译成大白话，“游戏规则要改变，不改变就去死”，这样的话，一切问题就迎刃而解，外部环境在变化，也不由得共产党不去改变游戏规则，而这些改变在我看来是令人欣喜的。中国会变得更美好，更适合人居。

1.最底层的广大人民生活质量会提高，将逐步推进西化，医疗教育负担会逐步变化，可以预见，医疗体系、教育体系会进行渐进的改革，目前这两大问题已然成为了，社会的大毒瘤。医生、教师的职业道德素养需要提高了。
2.工业化进程会提高，劳动密集型逐步会减少比例，当然是很缓慢的渐进。
3.民族思维会更活跃，其实这个方面是共产党在打压的，但前进的步伐不会停止，现在有些理解西藏和台湾了。
4.政府的行政能力会有所提高，这个方面我个人是相信人的本质还是有职业诉求的，腐败是不可避免的，但阳光性是这个社会的本性，而如今的腐败，不是个人的问题，而是整个社会的问题，目前的社会情况下，民众都很黑暗，但随着整个社会诉求的提高，政府的执政也会提高。

唯一阻挠社会进步的问题就是一党，一直我还是比较拥护一党的，只要领导人有些良知，可以减少一部分多党造成的社会资源浪费（有些时候在野党的行为纯粹就是扯淡），但如今，社会资源已经极大丰富了，也许需要改变档位了。当然只要一党执政能力依然强悍，这个需求并不急切。记得前几天看到的一个牛人说的一句话“整个80年代，亚洲很多国家完成了民主化进程，包括蒋经国、泰国等，而中国却错失了这个良机”，毕竟我还年轻，不清楚那时候的环境如何，不过在我看来，那时候的大陆也许在经济实力上都不支持民主化，不清楚泰国是怎么进行的，不过现在的民主泰国，看起来也很悲剧。

无论如何，对于中国的长城，我是强烈不满意的，鉴于此，在我经济实力允许的时候，真想移民逃跑。中国人太多了，真该政府性的移民几亿人到俄罗斯、加拿大、澳大利亚去。
嘿嘿，看来我具有逃跑的潜质。
