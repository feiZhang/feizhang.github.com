一个插件引起的血案

chrome的插件 html Validator 会自动重新请求一边你访问的页面，从而导致 服务器的session由于这次请求而改变。导致代码执行错误。
找到问题的途径：
观察现象：

  1. 在本机有问题，但是在其他开发人员的机器没有问题。 （确认代码没有问题）
  2. 在本级有问题，上传到服务器，本机浏览器测试没有问题
  3. 正常浏览有问题，但是右击显示代码，之后刷新，就不会出现问题。
    解决问题：
    第二个现象是导致解决不够迅速的根本：（这个还没搞明白为什么）
    如果是浏览器问题，应该上传到服务器也有问题出现的，但是服务器没问题，所以判断为本机Web服务器配置问题。
    多方查找本地配置，还找不到问题，而且个人感觉不是问题的原因
    随想到可能是ajax多次请求的问题，将代码简化为无ajax，问题依旧

最终找到问题的原因是现象3：
所以很疑惑，就右击页面->审核元素，发现chrome的插件，追加了不少html代码，遂一个一个的禁用插件，最终确认是html Validator引起的原因，然后就查看html Validator附加的js代码。发现确实会有一个xmlHTTPrequest请求，重新获取了一遍页面。。

var xhr = new XMLHttpRequest();

// If the result is finished, send complete page HTML code to W3C validator
xhr.onreadystatechange = function () {
if (xhr.readyState === 4) {

var htmlForm = document.createElement("form"),
htmlInput = document.createElement("input");

htmlForm.action = "http://validator.w3.org/check";
htmlForm.method = "post";
htmlForm.enctype = "multipart/form-data";

htmlInput.type = "text";
htmlInput.name = "fragment";
htmlInput.value = xhr.responseText;

htmlForm.appendChild(htmlInput);

document.body.appendChild(htmlForm);
htmlForm.submit();
htmlForm.parentNode.removeChild(htmlForm);
}
};

// Send XHR request to itself to get the entire HTML code
xhr.open("GET", location.href, true);

最终的疑惑：为什么同一个浏览器访问本地和服务器，服务器就没有问题（本地是host域名，服务器是真实域名）,怎么想都不应该的。

经验： 浏览器的插件有可能导致程序的异常。尤其是个页面追加js的插件。

忠犬八公的故事

看《忠犬八公的故事》这部电影的时候，里面的日本人说：“秋田犬是不会捡球的，不会因此去取悦人，如果他那么作了，那是因为其他的原因”，后来小八为了拉近于帕克的距离，确实去捡球了，我认为他应该是在告诉帕克，“我很重视你，我会作你让我做的事情”。“狗腿子”这个词，也正是指出了：某些狗为了得食，取悦于人。

这个故事，让我想到养儿子这件事（也许一辈子只能养一个儿子了，没事老爱想想这事），儿子小，不明白，但是在我们对儿子有求必应的时候，应该要非常明白的告诉他，对他好，是因为：“爱他，希望他好”，而不是在纵然他、迁就他，让他仗着对她好，就任性、忘形，让他知道，爱是相互的。。要分清楚，同样是捡球，意义缺是不同的。儿子小，不能自己明白，所以我们需要明确的告诉他，也许能加速他的理解。

Sigma Grid的一个bug

测试的例子：test.html
<input type="text" name="id" id="jjid" style="width: 200px; ">
<script type="text/javascript" charset="utf-8">
alert(window.jjid);
</script>
则在 firefox中显示为undefind,而在chrome和safari中能够获取到input的DOM对象。
则在sigma的使用过程中，如果页面包含一个input的id="id"则，直接出错。
例如：test.html
<script src="gt_base.js" type="text/javascript"></script>
<input type="text" name="id" id="id" style="width: 200px; ">
则会在chrome的调试控制台中出错，无法找到"unload"方法。

自定义程序员等级

  1. 掌握基本的编码语言，了解基本的SQL语句
  2. 了解系统、WEB等基础运转原理
  3. 熟练使用某种编码工具，提高编码效率
  4. 熟练使用源码管理工具
  5. 基本的调试能力
  6. 了解基本的设计模式，能够写出有条理的代码，并能够重构自己的代码

  8. 编码牛人，能够实现你所想到的所有功能
  9. 能够针对一个产品，提出自己的大体实现思路，并能设计产品的各项功能（界面、操作、运行等）
