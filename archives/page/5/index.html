领导者个体的延伸 公司

原来一直认为，企业管理就是对人的管理，或制度化，或人性化，近来发现，其实不是老板对员工的管理，是老板对自己的管理，员工是领导自身的延伸，将自己的一头两臂，延伸为三头六臂，甚至更多，从而完成一些个体无法完成的事情。

公司的性格继承老板的性格，宽容的、开放的、细腻的、谨慎的等等，公司的行为倒是老板行为的映射，这是一个笼统的概念，人偏瘫了，也管不了自己的手脚。而且手脚自身，也有各自的特点，我们期待自己的手脚有力，那就需要经常的锻炼它。

那么，一个成功的企业，就是一个将自己的优点放大到所有员工身上，而自己的缺点，尽量限制，而无法根除。

那么，你是一个什么样的人？你想问你的老板一些什么问题？

你的理想是什么（分阶段）？怎么实现？核心做法？有没有深入底层员工？你的核心优点是什么？想放大什么特点？你的最大缺点是什么？怎么弥补？成长的关键点是什么？怎么应对？

好吧，让我来积累下这些问题，也许能有一张测试题，不准确却大体上能够度量你的领导能力。

2012-05-09

养老机构研发管理失败的总结

需求只是收集，没有分析

这个项目相对很简单，所以需要分析的内容不是很多，但是在前期依然没有将整个项目系统地进行分析，导致后期对于补贴处理的不明确，虽然不影响进展，但不能保障完成一个精巧的系统 一个重大的失误：没有全面查看用户提供的全部资料

Demo设计不够具体，没有进行反复论证

同上，不是很差，但也称不上为好

技术积累

数据列表、基础信息维护、简单数据统计、页面框架 尚未模块化、标准化

时间问题[无法控制,有点烦]

由于时间的压缩，导致所有工作的提前介入，各方面不能够精益求精

人员沟通管理

各部位人员，并没有在思路上对项目的进展达成一致的共识，没有反复确认的过程，依然还是原来埋头苦干的工作模式

综上所述，目前虽有改善，但依然是原来的老路子，将工作处理为体力劳动，而没有真正转换为脑力劳动。 成功的标志：大量的时间应该花费在，思考、设计上，而非敲键盘上。

2012-07-10

苹果带给我们的，并不是几个好产品，而是全心投入做出好产品的价值观。

并未打算此时此刻就要做的多么多么的好，只是想让我们的产品，至少是团队智慧的结晶，而不是一坨坨生硬的代码

这个事情是很困难的，每一个人都会有各种各样的想法，还会有各种各样的问题。

  1. 有各种各样的设计思路，怎样取舍？大多数通过投票解决，主要把关人员强烈坚持的，可以独裁[1.角度不同，开发人员可能会选择容易实现的 2.阅历和知识不同，感悟不同]
  2. 项目周期的把控，尽量争取宽松的开发环境

一些附带的问题：

  1. 不要过于直接的否定别人、批评别人
  2. 要多否决的原因做出必要的解释
  3. 要将此事与个人的发展之间的关联，明确说明

所有事情，要解决的最大的一个问题：为什么我要努力作？

  1. 我要通过项目体现自己的价值，获得成就感、自豪感、荣誉感
  2. 我要通过项目提高自己的能力、知识、品味
  3. 这是一个待遇优厚的工作，我不能丢掉这个工作（这个目前难以实现，且持有这个理念的人一般不会是什么高人[特定情况例外]）
  4. 挑战困难的心理，超越自我是一件有快感的事

是不是还有其他的方面能够调动员工的积极性？

最终做出的一些部门决定：

  1. 所有人员都应该有制作优秀产品的想法
  2. 所有人员应站在客户的角度来理解产品的操作
  3. 美工对界面美感作全局把控，对于有异议的设计思路，可以发起讨论如何实现
  4. 每个人员都可以对任何事情发表个人看法，并反馈给相关人员，如有冲突，可发起讨论
  5. 尽量拉长设计周期
  6. 每个人都可以为自己的思路争取资源，但也要适当妥协

2012-10-19 初级思路

年初开始接手部门管理工作，回顾一下，已经经历3个阶段。

第一阶段：熟悉了解缓冲

  1. 熟悉团队成员的做事风格，个性
  2. 让团队成员熟悉自己的风格
  3. 缓冲,领导更替的变化

总结：现在看来，熟悉这个阶段做的相当不到位，其实和结婚一样，不在一个屋檐下，很难去了解其中滋味，还是需要一起做项目，一起写代码才会真正的了解

第二阶段：松散式管理

个人性格的体现，总是期望：个人能力佳，有一定的责任心，有向上动力的团队，偏重于对积极性的调动，有点脱离现实。

  1. 2\8分的理论已经证明，除非精挑细选，只能有一小部分成为精英
  2. 现实的状况是没有多少层面来极大激活积极性

同时：

  1. 对于旧员工的管理过于放松

好在：

  1. 总算是积累了一部分技术资源
  2. 客户意识极大增强
  3. 团队向心力较好

第三阶段：模式化开发

既然无法发挥每个人的个性，那就按照传统套路来吧。

  1. 基于前一阶段的技术积累，深度定制基础模块，为项目开发提供大量支撑，减少工作量
  2. 基于现实情况，推动项目管理继续完善。

目标：

将项目开发，固化为一个标准流程，不用大量思考，按部就班地高效整合资源，快速迭代完成项目开发。

最终总结：

  1. 破釜沉舟：看起来很牛，但只能作为奇招，治国安天下，还要传统着来
  2. 拔苗助长：看起来很傻，现实中，大家却都很着急
  3. 捡了西瓜丢了芝麻：社会极大的丰富，让人延缓缭乱，合适自己的鞋子才是好鞋子
  4. 水滴石穿：水或许都没想到能把石头滴穿，就这样日复一日的平淡中，也会创造奇迹

无论如何：按照自己舒服的方式来生活，没错，在你能选择的范围内。

海底捞你学不会 笔记1

海底捞成功的奥秘在哪里？我认为黄铁鹰的总结重点在一段话：养而不爱如养猪，爱而不敬如养狗。而人呢，只给吃和爱是不够的，还需要尊敬。什么是对人的尊敬？见老板鞠躬给领导鼓掌？那是对地位和权力的尊敬。对人的尊敬是信任。信任你的操守，就不会把你当贼防；信任你的能力，就会把重要的事情委托给你。人被信任了，才会有责任感。而信任的唯一标志就是授权—海底捞给予火锅店的普通员工物质回报，还给他们“信任”与“授权”，让他们一同收获幸福感和成就感。

信任不是说出来的，而是做出来的。张勇在海底捞公司的签字权是100万以上；100万以下是由副总、财务总监和大区经理负责；大宗采购部长、工程部长和小区经理有30万元的签字权；店长有3万元的签字权。书中说，这种放心大胆的授权在民营企业实属少见，但我认为这都不是最重要的授权，海底捞最重要的授权给予了基层的服务员：不论什么原因，只要员工认为有必要，都可以给客人免一个菜或加一个菜，甚至免一餐。

这个小细节体现了海底捞管理的奥秘。从服务员一手干起的老板----张勇明白：一个餐馆不论其名气或者装潢，客人从进店到离店，始终只跟服务员打交道，所以餐馆客人的满意度基本掌握在跑堂员工手里。怎样才能服务好客人？那就要善用这些在现场的普通员工，多发挥他们的才智。做法很简单，授权，给他们作决定的权力。黄铁鹰总结说：如果客人对你餐馆的服务不满意还要通过经理来解决，这个解决问题的本身又会增加顾客的不满意度。

一般餐馆里，顾客结账时不会同服务员谈打折优惠。为什么？谈了半天，那个忙得跳脚的服务员连是否能给个98折优惠都闪烁其词，因为她要看大堂经理的脸色。这种折扣，给与不给，顾客与餐馆都双输—顾客找经理要到折扣，也不会念餐馆的好。

这等于海底捞的服务员都是经理，因为这种权力在所有餐馆都是经理才有的。德鲁克认为，企业的员工是否是管理者并不取决于他是否管理别人，所有必须坚持自己的目标和标准，进行决策，并对组织作出贡献的员工，实际上都在行使管理者的职责。显然，在海底捞的管理体系中，每一个基层服务员都是一个“管理者”，对服务品质起到关键的影响，对公司至关重要。

为什么要推荐 Clear

从 Apple4.us 作者：Lawrence Li
Clear 这款 iPhone 上的 to-do 软件在 UI 设计上的大胆为近年罕见。它打破了苹果在《iOS 人机介面规约》中设定的大量原则，采用了类似游戏式的浸入式全屏介面，全面放弃按钮和各种「栏」，所有操作用扫、拉、捏、括等手势完成。这种破局式的设计让人耳目一新，但也增加了学习成本。 我并不认为 Clear 的设计完全成功。这不是因为它缺少通知提醒功能或任何其它功能：缺少的功能要么在未来会加上，要么不会，但无论怎样，功能的多少或有无不是评价软件的唯一标准。 事实上，我认为 Clear 的手势略显复杂，且不够直观。例如「捏 / 括」（pinching）手势既有「退回一个信息层级」的功能，亦有「添加一个 to-do 事项」的功能，这令我在使用中产生过不少困惑。 但我们需要 UI 设计界不断出现新的标杆，需要那些不去担心「会不会因为违反了人机介面规约而被苹果拒之于 App Store 门外」的设计师。和把软件设计得直觉、易用、符合使用习惯相比，这是更高的目标，也是更危险的策略。但正是这些冒险家推动着 UI 设计艺术的前进。 听说 Clear 在美国 iTunes 店只用了九小时就爬上了收费软件排行榜榜首。我希望它在中国 iTunes 店也能有更好的成绩（目前是第八名）。如果最前沿的设计未能引起我们自己的用户和 iOS 开发社群充分的关注、研究和讨论，我们就很难看到像 Clear 这种充满 chutzpah 的作品出现。 Clear 的购买地址在这里，目前限时优惠价人民币六元。 不一定这个产品一定成功，但是却不能磨灭他的功绩，可是事实缺并不那么简单，很多时候，我们无法做到纯粹，有各种各样的妥协、无奈，什么才是衡量所谓成功的标准，事实？恐怕未必，看来一切都皆在我心，管他妈的什么世道，一个世道的所谓成功，就是一个人的成功，管他是否民主，就如邓公，到现在很多人来否定他，但是又如何，不管事实如何，煞笔永远存在。无愧我心足以，那么!赵括呢？马谡呢？ 靠。这是什么一个轮回，难道都是扯淡？
