关于丁丁的病我要崩溃了

输了5天的抗生素，3天的时候都好了，按照医生的话，我们还多输了2天，可是没过一天，丁丁又发烧了，他妈妈急的热锅上的蚂蚁，而我更是无奈而又悲愤。

因为丁丁的病，进来在浏览一个德高的老中医（不慎熟）的论坛，里面对于抗生素的控诉，让我恐惧（而现实又很无奈），所以极力说服夫人，不要轻易去医院看病（因为到医院，基本的套路就是随便问问病情，然后就是验血，然后就是血象高，然后就是输液（抗生素），最终的结果还是，过几天又反复）其实我一点都不排斥西医的，而排斥的仅仅是现如今的医德、医术、就医环境。
医德：能详细询问病情都算不错的了，即便是装
医术：这个很好理解，在我们IT界，高手也是寥寥的，更多的都是按章办事，医生中有慧根的也应该不多，甚至现实中，愿意主动用心提高自己医术的也不多，所以嘛，诊断也就是个大众化招式。
就医环境：这年头，某些人到医院，你不给他输液他还心里不爽（应该不是多数，大部分人还都知道是药三分毒的），虽不至如此，可一旦自己的子女病了，那有病乱投医的劲都提到嗓子眼了，干什么都要立竿见影，另外就是现在的中国，大家的经济建设欲望都太强了，当然也包括医生，毕竟他们没达到富人层次，他们有很多的经济追求。

鉴于此，求人不如求己，自己多了解些医学常识，可是很多时候，很多论点都是相互冲突的，比如，这位德高的老中医的说发烧就是身体在自愈，在强化自己的免疫系统，基本可以无视，主要精力不放在退烧，而放在调理身体，而又有更多的人号称，发烧必须24小时内退烧，按照我的理解我是支持前者的，可是大环境不相信，我也无能为力，毕竟我不是医生，更不是公认有成就的医生，所以我能有的，就是看着悲愤，为儿子祈祷，只能拿着现在的医学已经发展到这个阶段，人类不用有自己的抵抗力，而可以完全依赖外部的药物，要么我能怎么样呢，是呀，我可以自学医学，可是到什么时候我才能学会呢？我拿什么做验证呢？我有的只能是无奈，而且我更相信的是中医，而一旦我了解的更多，我对西医的抵抗也就更强，可是现实中，老婆、父母是不会给我这个机会的，他们一心关心的就是退烧，烧坏脑子。

最终的结果可能还是要随大众，因为舆论的能力太强大了，而且很多人不愿意看着你和他们不一样，甚至不愿看到，你儿子不去医院就能治好发烧，甚至会鄙视你不够爱你孩子。
如果我们活着不能独立思考，我真不知道活着还有什么意思。
小丁丁现在看起来有些好转了，总结起来，其实什么药都没吃，虽说吃了一副泡出来的中药，其实我真没把他当成什么妙方，仅仅是廖胜于无，看着小丁丁还挺精神的样子，我坚信小丁丁能够自愈，因为他没有任何其他症状，仅仅是发烧，如果有诸如红肿、起泡等现象，我会及时就医的，哪怕我知道去见的是一个庸医。

一直有一个想法，那些医生那么想过美好的生活，为什么不提高诊疗费用，而在药上赚钱呢？也许他怕舆论，他更愿意安全而隐蔽的赚钱，毕竟这也都成了行规，而且外人很难质疑，每个人有每个人不同的治疗手段，药量的大小各自有各自的理解。

即便如此，我们还有改进自己治病的方法，比如：如果不是很紧急的病，我们可以去多个医院就诊，而先不用药，权衡那个说的更有道理，就用谁的药，虽然有些麻烦，但为了健康，应该是值得的。对，以后可以试试，毕竟现在医院很多，就当撞运气吧。

一个js的奇怪错误，在unix系执行

最近项目总是出现一个js的奇怪错误，在公司的服务器部署，就没有问题，而放到公网，就会出现莫名其妙的错误，比如:break不行等等，按照ie提示的位置去看，代码是没有问题的，firefox下没有问题。

偶然间发现，可能是文件编码的问题，在我机器上eclipse打开，中文乱码，但是这些中文都是被注释掉的，随改变编码，复制，将文件修改为项目相同的编码gbk，粘贴，再部署，居然问题就没了。。

看来在unix+was下，js如果被认为编码不符，也会让ie执行不正常，即便是注释，这个问题很诧异。

小丁丁病了

小丁丁又发烧了，近来感觉小丁丁总是病，几乎每周见他的时候，都在吃药，心疼的很，尤其是输液时哭闹的样子，其实这些也不算什么，有病自然要医的，主要还是怕这样下去小丁丁的免疫系统会不会被药物给搅乱，形成不了自己的良好免疫系统，不过担心也没什么用，一方面是：只要他一发热，她妈妈就着急上火，直接就想退烧，立马治好，按照我的思想是，稍微烧烧是没问题的，而且按照一些说法是发热就是在完善抵抗力，另一方面是：现在的医生，就知道用抗生素，不知道是为钱？还是目前的医疗条件就是这样，应该两者都有。

因为小丁丁的病，已经和老婆闹了2次大别扭了，如果不是为了丁丁，估计我是要放弃目前的生活了，就算是我自己的问题吧，那我总有选择退出的权利吧，不过既然无法退出，那么还得好好的过，现在的我是真没办法，也没有太多的心情来调整了，主要问题还是在于父母和妻子的问题，而我和妻子的问题，目前的我，只要不出意外，只要不牵扯其他人，我完全可以默认掉，而对于父母我真是不知道该怎么办了，我没能力没权利去劝解没读过多少书的父母去改变，也没能力去改变妻子的想法和做法，能做的只是袖手旁观，瞎感慨。谁知道以后会过成什么样，只能认之为“天命”了，问题最大的是，他们的争执，或多或少都会伤及到小丁丁，很是令人郁闷，无奈无奈，每此时总是想起当年聪明一休里的一个小故事，找出亲生父母的办法，就是让他们用小孩去拔河，而放手者才是最爱她的亲生父母。

产品的质量和用户的体验

植物大战僵尸已经火了好长一段时间了，只是因为自己工作太忙，而没有实现上手，最近见弟弟玩了，自己也试了一把，很不错，唯一对我来说不爽的就是太easy了，不过我还没通关，不知道后面怎么样，按照弟弟的说法是，一把通关，估计是比较简单点，导致使用一个战术可以基本应付大部分的关卡，从而仅仅是个休闲游戏，而非一个智力游戏，导致其生命周期不够长，如果考虑加入一些AI估计会更好些。。总之算是个很不错的游戏，总结下优点：

  1. 界面清新、简单上手
  2. 普通关卡中夹带特殊关卡，有利于改善游戏体验
  3. 比较丰富的植物元素，可玩性更高
  4. 幽默的人物和货币、商店加入，增加了可玩性

  5. 休闲里面有智力元素，增加可玩性
    不足：

  6. 操作要求不高，本身就是休闲游戏，当然不必太高的操作要求，不过可以提供基本和高级版本，以满足高端玩家

  7. AI太低，难度太低，智力因素过少，不利于进入研究阶段，从而影响声明周期
    最近公司又上了一个项目管理软件，redmine，公司将所有员工的邮箱设置为了139信箱，则测试每提出一个bug会给开发人员一个邮件，从而得到一条短信提示，结果，某人一天得短信50多，结果就导致其对这个工具第一印象相当不好，估计推广就有点阻力了，当然这个问题不是���件的问题。

上面两个软件，给我的启示就是，如果想做个好的产品，用户体验必须做好，需要多思考，用户没有一个良好的体验，绝对造就不了一个优质的产品，思考再思考。

最贵的几条鱼被我养死了

前天刚买了2条清道夫和富贵鱼，今天全死了，小金鱼也死了一条，省9条了，总结问题：

  1. 由于小乌龟一直不吃东西，就在鱼缸多投放的食物，估计都被鱼吃了，而且还有剩余，污染了水
  2. 为了小乌龟吃食，打开了加温器，将温度提高到30度，加速水质变坏
  3. 过高的估计了清道夫的能力，结果鱼缸里的粪便没有即时过滤掉
  4. 也许鱼太多了，10条小金鱼+2条富贵鱼（大金鱼）+1条清道夫+2只乌龟
    此前只养了10条金鱼时，半月不换水也没事，一周才喂一次（经常不在家），这次才一周，水质都变坏了，可怜了小鱼鱼门，下周还得再去弄2条清道夫来。
