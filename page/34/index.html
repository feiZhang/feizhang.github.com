锻炼身体

一直都有锻炼身体的必要，但是都未真正执行，最近耗巨资购入一辆山地车，一方面上下班坐骑，另外就是可以依次来进行体育锻炼，这两天还不错，每天至少骑上个一小时，小冒汗，最高时速达32（不知道码表是否精确），晚上回家也不再继续pc了，没事做几个俯卧撑，转转腰呀头呀，感觉挺好，同时也发现自己真是不够健壮，没有个能见义勇为的好身体，估计自保都困难，10个俯卧撑都很难，而且腰部是在虚弱，保持下俯卧撑的姿势都坚持不下去，需要长时间的锻炼了。。

兄弟，这次一定要坚持下去呀。

Ubuntu下安装Xdmcp服务，使用Xmanager接入

ubuntu下安装xdmcp服务（即远程主机通过xdmcp协议连接到ubuntu的图形终端，以图形终端方式登录）： 1. 登录本机图形终端（本机按Ubuntu/Gnome为例设置） 2. 选择system->Administration->logon windows切到remote项，修改其中style为same as local,如为中文菜单请相应选择。 3. 选择system->Administration->logon windows切到General项，修改其中Disable Multiple login for a single user前面的X去悼,如为中文菜单请相应选择，否则会出现能显示画面但无法登录（输入用户和密码后又回到登录画面）。 4. 修改sudo vi /etc/gdm/gdm.conf（建议先将gdm.conf备份，以免无法启动时，可以快速恢复） 其中[Daemon]中#RemoteGreeter=/usr/lib/gdm/gdmlogin中前面的#号去悼； [Security]中#DisallowTCP=true中前面的＃号去悼，并将true改为false; [xdmcp]中#Enable=false中前面的＃号去悼，并将false改为true; 5. 修改sudo vi /etc/inittab增加一行id:5:initdefault:表示系统将自己运行在多用户级别５的模式下； 6. 重启sudo reboot，不建议sudo gdm restart这样gdm并未生效 7. 检查服务器和客户端的网络防火墙配置，保证177端口和6000-6100端口可用； 8. 使用Xwin的服务器程序连接ubuntu图形界面即可；

昨天去体检了,鼻炎咽炎,其他挺好

昨天参加了公司安排的基本体检，顺便自己有附加检查了鼻子咽喉，一直以来：咽炎、鼻炎、脚气是我郁闷的3大顽症，而且对身心感受非常不好，虽然不是什么严重的病，但是十分影响生活质量，就如，鼠标之于电脑。

今决定，手术之，随发现自己的经济状况不甚好，虽然待遇也不错，但是支出情况确实比较严重，自己也纳闷，为何会有如此高的日常支持，看来今后需要做下统计了，而且被逼无奈也得改变下自己的生活消费方式了，估计手机、电脑类非必要，而且非自己愿望的面子东西和不慎好玩的娱乐品都不要在购买了，经济能力有限，就只能缩减自己的兴趣范围，今年的消费就此停止，如果有机会顶多安排一次2000元以内的旅游，其他时间就花在自己的17-80(a200套头)上得了，待明年看情况再安排计划。

其实在有医保、养老保险情况下，我们也真是没必要做太多的储蓄，孩子教育？除非有较高的择校费，应划入日常开支。唯一不保险的就是重大疾病，其实对于我自己来说，重大疾病=死亡，可以忽略，但，人总是有很多挂念，希望商业保险尽快发展，以填补此方面空白。

对于个人的政治前途、生活，每个人也有自己的选择，比如：

你现在手里有10万，你可以做两件事的其一：买房或等待机会上进，一般情况只能是二选一，要么你在现在就"退休"过上凑合体面的生活，要么你给自己一个争取一个机会，找到新"工作"未来体面生活，这就是你的选择。

Wordpress Atom协议的使用

最近想用wordpress最为一个数据展示系统，需要将数据导入到wordpress系统中，网上找半天，未发现有人对此做过应用及介绍，只好自己来研究了。 通过wordpress.org 了解是wp-app.php 文件提供了atom功能，打开源码，就可以找到其支持的各种方法，POST、GET、DELETE等，如下：

$this->selectors = array( '@/service$@' => array('GET' => 'get_service'), '@/categories$@' => array('GET' => 'get_categories_xml'), '@/post/(\d+)$@' => array('GET' => 'get_post', 'PUT' => 'put_post', 'DELETE' => 'delete_post'), '@/posts/?(\d+)?$@' => array('GET' => 'get_posts', 'POST' => 'create_post'), '@/attachments/?(\d+)?$@' => array('GET' => 'get_attachment', 'POST' => 'create_attachment'), '@/attachment/file/(\d+)$@' => array('GET' => 'get_file', 'PUT' => 'put_file', 'DELETE' => 'delete_file'), '@/attachment/(\d+)$@' => array('GET' => 'get_attachment', 'PUT' => 'put_attachment', 'DELETE' => 'delete_attachment'), ); 然后就是用curl 进行一个一个测试，目前我不知道windows是否有相似的工具，我是在ubuntu下进行的测试，经过千辛万苦，终于将基本的操作方法搞明白了。 重要的两个方法 1.DELETE 删除一篇文章 curl -v -X DELETE -u admin:333 http://liangpeng.net/wp-app.php/post/3 其中的-u指定用户名密码 2.POST 创建一篇新文章 curl -v -X POST --data @t.xml -H "Content-Type:application/atom+xml" -u admin:333 http://liangpeng.net/wp-app.php/posts/ 其中-H是设置http请求的头信息，在服务器端会进行Content-type类型验证，所以必须增加此信息 其中@t.xml是一个文章内容的xml文件，可以通过 http://liangpeng.net/wp-app.php/post/2 获取到一个真实数据，修改下就能用了，我是直接把多余的信息全部删除，只剩下标题和内容字段，测试有效。

目前两个操作基本满足我的需求，关于Get，直接使用 http://liangpeng.net/wp-app.php/post/? 既可以获得，很是简单，当让也需要用户名密码。

注意：

  1. wp-app.php 只接受/post/(/d)+格式的url重写，如果你的服务器不支持，请设置，目前apache下貌似使用wordpress自己生成的.htaccess即可，nginx我还没搞定
  2. 之前一遍英文文章(http://torrez.us/archives/2006/08/21/486/)介绍的使用cookie登录，貌似现在已经不行了，wp-app.php使用的是$_SERVER获取的User变量，curl 可以通过参数 -u 直接传递。
