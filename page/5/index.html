Ubuntu搭建time Machine备份的问题

1.系统问题，ubuntu apt-get 安装的netatalk 不能读取配置文件，查看/etc/init.d/netatalk 可以看出，配置 /etc/netatalk/afpd.conf根本没用，这是最严重的问题，看来不能太相信一些不该出问题的东西

2.登录问题。必须设置成能游客登录，才行，用系统帐号登录，总是登录不成功（目前还没解决），而且提示的是“请检查服务器名称或 IP 地址，然后再试一次。如果问题持续发生，请联系系统管理员。”，让人很疑惑，一直以为是网络配置问题，此时输入错误密码，会窗口晃动，表示登录失败。 -U uams_guest.so 是启动游客登录功能。

其他配置就很简单了，跟网上搜到的教程基本一致：

/etc/netatalk/AppleVolumes.default
/home/back “TimeM” options:tm

/etc/avahi/services/afpd.service

<?xml version=“1.0” standalone=‘no’?>
<!—-nxml-—>
<!DOCTYPE service-group SYSTEM “avahi-service.dtd”>
<service-group>
<name replace-wildcards=“yes”>%h</name>

<service>
<type>_afpovertcp._tcp</type>
<port>548</port>
</service>

<service>
<type>_device-info._tcp</type>
<port>0</port>
<txt-record>model=Xserve</txt-record>
</service>

</service-group>

版本：ubuntu 12.04 server afpd 2.2.1 avahi-daemon 0.6.30

网上说的IPV6、cbd，貌似都不是问题；

项目管理进入沼泽地区

如果说原来部门制度的建立，还算进展顺利的话，接下来的进程，将漫长而艰巨，因：这是一个意识形态的改变
苹果带给我们的，并不是几个好产品，而是开辟了一个尊重用户体验，微创新的新局面，现在的IT业，哪个有点意识的，不把精力放在用户体验上，能不能做好是一回事，想没想是另一回事，就如：小米如此成功，还是被老罗批无用户体验一样，这个东西并不是你想有就有的
并未打算此时此刻就要做的多么多么的好，只是想让我们的产品，至少是团队智慧的结晶，而不是一坨坨生硬的代码
这个事情是很困难的，每一个人都会有各种各样的想法，还会有各种各样的问题。
1.有各种各样的设计思路，怎样取舍？大多数通过投票解决，主要把关人员强烈坚持的，可以独裁[1.角度不同，开发人员可能会选择容易实现的 2.阅历和知识不同，感悟不同]
2.项目周期的把控，尽量争取宽松的开发环境
一些附带的问题：
1.不要过于直接的否定别人、批评别人
2.要多否决的原因做出必要的解释
3.要将此事与个人的发展之间的关联，明确说明

所有事情，要解决的最大的一个问题：为什么我要努力作？
1.我要通过项目体现自己的价值，获得成就感、自豪感、荣誉感
2.我要通过项目提高自己的能力、知识、品味
3.这是一个待遇优厚的工作，我不能丢掉这个工作（这个目前难以实现，且持有这个理念的人一般不会是什么高人[特定情况例外]）
4.挑战困难的心理，超越自我是一件有快感的事
是不是还有其他的方面能够调动员工的积极性？

最终做出的一些部门决定：
1.所有人员都应该有制作优秀产品的想法
2.所有人员应站在客户的角度来理解产品的操作
3.美工对界面美感作全局把控，对于有异议的设计思路，可以发起讨论如何实现
4.每个人员都可以对任何事情发表个人看法，并反馈给相关人员，如有冲突，可发起讨论
5.尽量拉长设计周期
6.每个人都可以为自己的思路争取资源，但也要适当妥协

不能作自己的一些影响

现实中，有些因素，往往会让我们不能按照自己的意志来处理自己的事情，总结了一些简单明了的影响。

  1. 扭曲个人心智，影响个人品格完善 ［总是不如愿］
  2. 影响个人行为对这个社会的试探性，推迟对个人行为的把握度，不利于自己优势的拓展
  3. 影响圈子人员对自己的认知度，从而影响自己的发展
  4. 不能很好的处理大家庭的关系，从而影响自己自己最大的根据地（作为心灵归宿的意义更重要一点）
    这样说来，一个疑问有点隐隐显现。老爸为什么这么弱势————老妈太强势了。
    有些事情我们无法改变，那就顺其自然，但是尝试还是可以有的。

领导者个体的延伸-公司

原来一直认为，企业管理就是对人的管理，或制度化，或人性化，近来发现，其实不是老板对员工的管理，是老板对自己的管理，员工是领导自身的延伸，将自己的一头两臂，延伸为三头六臂，甚至更多，从而完成一些个体无法完成的事情。

公司的性格继承老板的性格，宽容的、开放的、细腻的、谨慎的等等，公司的行为倒是老板行为的映射，这是一个笼统的概念，人偏瘫了，也管不了自己的手脚。而且手脚自身，也有各自的特点，我们期待自己的手脚有力，那就需要经常的锻炼它。

那么，一个成功的企业，就是一个将自己的优点放大到所有员工身上，而自己的缺点，尽量限制，而无法根除。

那么，你是一个什么样的人？你想问你的老板一些什么问题？

你的理想是什么（分阶段）？怎么实现？核心做法？有没有深入底层员工？你的核心优点是什么？想放大什么特点？你的最大缺点是什么？怎么弥补？成长的关键点是什么？怎么应对？

好吧，让我来积累下这些问题，也许能有一张测试题，不准确却大体上能够度量你的领导能力。
