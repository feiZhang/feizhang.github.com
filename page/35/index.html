一直弄不清出的一个经济学问题

今天看电视说日本人均收入4.8万美元，有点诧异，国内经济已经如此繁荣了，难道我们的中产阶级还比别人的平均差数倍？

一直以来也同意部分人说的所谓人民币的购买力问题，比如：同样购买一个苹果，在欧洲用欧元买可能是1欧元，而在国内用人民币买可能是1人民币，这样来说对于国际汇率，人民币的购买力是高的，而对于一些高科技涉及进口的东西，大家的价格就差不多了，所以在完全以美元计算世界人民收入的时候，国人是有些吃亏的（当然我不是说中国已经足够发达，而是说在表面上国人应该不致于那么落后的)。那为什么会造成这样的状况呢？是一部分商品不适合国际流通？那么原材料呢？有些搞不懂！

如果这些都是事实，那么人民币的汇率将如何发展？继续对美元升值？最后1:1美元，同样一个苹果=1美元=1人民币？ 还是继续通货膨胀？一个苹果=7人民币=1美元？

有点迷惑，哪位给个简单的解释，谢谢了。

从别人文章学习到得一端评价测试用例的感悟

如果这个 bug 是人为失误，比如在 C 语言中把 == 写成 = 。那么重新出现在原来位置的可能性很小。（比之更复杂的情况也是。) 通常，一个好的编译器（对于静态语言）或一个好的代码静态分析器（对于动态语言) 比自动化测试更有效。

如果 bug 存在于软件复杂的上下文，由于整体设计失误，或是繁杂的模块依赖关系造成。对于单个模块的测试也就无效，而问题被彻底弄清然后 fix ，在软件的生命期内也就几乎不会再出现了。

归根到底，我们需要的是减低模块耦合度、提高内聚性。比如写更小的程序，用进程而不是线程解决问题，制定简洁稳定的 IPC 通讯协议，而不是定一堆的 interface ，等等来提高软件的可维护性。

在许多时候：企图写出覆盖所有分支的测试案例是在浪费人的时间，而 频繁、自动化运行测试套件，则是在浪费机器的时间，为地球多制造一些温室气体罢了。

怎样书写个人Blog文章

最近发现一个叫 张宴 的牛人的blog，此人岁数不大，技术却不错，看起来人品也可以（哈哈，现实中不认识），其blog更新不算快，但是貌似每篇都是其一字一字写出来的，当人有些东西也是综合其他人的一些技术，在现在的环境下谁遇到问题不google一下先人的方法，但是也都经过自己的重编辑达到内容的完整性，给人的感觉是对每一篇文章都细心的对待，使其完整的描述一个事件，当然我不反对把blog当成自己的资料夹，但是在copy的过程中如果是好文章，则排版好之后发布（当然不能少了转载字样），而有些自己感觉不拓的地方就要细心修正，使其达到能用的境地，比如：你用的是VISTA，而参考文章是xp，则其中处理的方法肯定有差异，则详细描述之。。

总结：认认真真写好每篇文章。

使用Netsh命令在Window下快速切换Ip

使用netsh命令却换ip设置，具体可通过netsh ？获取帮助，帮助写的很详细，下面是网上找的winxp下的脚本，其中的local貌似是本地连接名字。
在Win7和vista下，应该是这样（看见那个本地连接没？就是你的网卡的名字，用他来指定网卡还真有点晕）：记着要以管理员权限执行..
@echo ip197
netsh interface ipv4 set address "本地连接" static 192.168.0.197 255.255.255.0 192.168.0.1
netsh interface ipv4 set dnsservers "本地连接" static 192.168.0.1 primary
WinXP下
@echo ip_beijing
netsh interface ip set address local source=static addr=10.56.100.112 mask=255.255.255.0
netsh interface ip set address name=local gateway=10.56.100.254 gwmetric=1
netsh interface ip set dns local static 10.68.100.211
如果需要为IE设代理服务器（我上网主用Firefox，IE代理是为了msn），则可以加一句 call ipbj.reg此reg文件，比如
Windows Registry Editor Version 5.00
[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings]
"ProxyServer"="10.18.100.133:8080"
"ProxyOverride"="10...*;<local>"
"ProxyEnable"=dword:00000001
最后面的1代表启动，0代表关闭
