奋斗与不妄为

社会矛盾是避免不了的，不管政府再怎么调控，顶多的是对富人提高了开资，但是按照负担收入比，富人的生活质量还是远远高于穷人的，而想要舒服生活的唯一真理就是“奋斗”，走在大众的前面，即便全民大倒退，你还是好于常人的，也许这样讲过于自私，但是，对于每个人也都是有机会的，前提是你要奋斗，而非等着被“救助”，即便你的出身不好，你也可以在出身不好那群人中出众吧。

成功并不是很难的事情，只要简简单单的比大众更吃苦一点更有心一点即可，不过成功后的再失败也是很简单的，成功后最大的可能就是妄为，不知天高地后，以为自己有神佑，其实不然，稍不留心即会跌倒。

没有智力靠体力，不管怎样，都要动起来。 明年准备接点私活干干，目标其实很简单的，实现起来没难度。

关于各种代码引擎或平台的思考

以下所述的引擎也可以理解为平台，在这里不做具体的区分.

最近在做采集平台的工作，以前也做过此方面的工作，但是有些久远了，具体的思路变化遗忘了，当时使用的方法是：主体程序+采集插件的模式，每增加一个采集类型，则写一个新的类，对其进行解析存储等，具体这样做的原因应该是：

  1. 制作引擎太复杂，自己的能力也有限

  2. 每个采集模块结构太复杂，很难提供一个统一的处理流程 这次做采集平台，公司资源深厚，技术总监技术也挺牛叉，但是建设到目前的情况下，不得不说，做的好不错，但是仅仅是看起来很美，实际用起来问题真是大，问题如下：

  3. 每个采集类型差异太大

  4. 采集到的原始数据需要进行大量复杂的后期处理（可以通过调整引擎适应）

  5. 各个采集类型之间又有复杂的关联关系 诸如此类的复杂问题，导致目前的引擎，没遇到一个问题，都需要进行完善，导致代码庞大而烦乱，对于一直着手此项目的人，倒是好说，但是对于新入人员，就会导致其迷惑、遗漏，也许经过重构，我们可能会获得一个强大的引擎，但是感觉还是有些难度，而且即使实现了也应该不算优美，随思考如下：

  6. 市面上听到大都的引擎，应该是游戏引擎（我也是听说而已，自己没有真实经历，也是自己的想法，有错的地方请斧正或跳过），仔细想想，应该是一套API，是一套对directx和openGL进行重封装的api，以及对一些功能项进行封装，具体的调用还是很复杂的，所以还是需要编码人员编码的，而非配置出一款游戏。

  7. 即便是成熟的游戏系统，对于其脚本，也都是一门编程语言，具有复杂的逻辑处理能力，而非简单的配置
  8. 最近研究的drupal，也是一个很复杂的平台，但是想要对他进行自定义也是需要进行逻辑编码的 综合而言，我认为，没有所谓的，能配置即可适应新需求的平台或者引擎，至少应该有一个逻辑编码的接口，或者做成插件形式的也不错。

Drupal的学习总结

最近加大力气研究了下drupal，发现的确是个非常强大的系统，代码结构十分复杂，目前的学习内容如下：

menu钩子

  1. drupal中的url路径必须通过api进行定义，有几种方法：1.直接通过api定义 2.在 *.module文件中的 {modulename}_menu函数中定义
  2. 其实drupal的所有url参数除了业务参数外，其他的路径全部在系统设置时已经完全定义完成，比如：?q=admin/build/themes 就是定义了一个 admin/build/themes 这样的名字的路径，同时有一个对应的回调函数，每次遇到这个路径则调用此函数处理。当然这样的路径也支持通配符。

  3. 目前来讲我只知道在module中能定义 *_menu钩子，在主题中发现不行 主题系统

  4. 在系统选择主题时，会将主题系统的设置进行处理并缓存，1.按照模块去生成主题信息缓存，比如node.tpl.php第一个选择是node模块的，2.根据主题引擎对主题信息进行补充 3.根据自定义主题对主题信息进行缓存。

  5. 在主题系统中可以通过 phptemplate_preprocess_node 函数，增加新的模板文件选择。比如： function phptemplate_preprocess_node(&$vars) { if (drupal_is_front_page()) { $vars['template_files'][] = 'node-front'; } } 算是重新定义了一个node模板。
  6. 子主题继承基主题的内容，但是必须自定一个css文件，如2一样要增加模板选择则必须在此主题下有父模板（node.tpl.php）的存在（因为这个问题，让我调试了1整天程序），如果没有node.tpl.php，则系统直接向基主题寻找node的模板，而忽略了子主题定义的新模板（这里我认为是系统的一个问题，也许遇到问题时我们应该先去查看系统bug列表，也许里面有呢，只不过我的英文太烂，所以只能自己去测试，我不能保证是不是如此，但根据我的调试，认为确实如此）

09年的回顾总结(跟风一下)

09年最大的收获应该是了解了一个新领域--股市，虽然之前就有进入，但是那时候的了解，跟现在不可同日而语，这完全归功于我的一位同学、好友梁朝阳，呵呵，从他给推荐的几个blog，我了解了价值投资、趋势投资等内容，对于股市也有了新的认识，总体接受了这个领域，09年也仅仅是肤浅的了解了一些概念，根据他人的经验收录了一批股票，个人收获总结如下： 1.要买有价值的股票 2.要审时度势，把握大势 3.没事不要贪心（目前还没做到），没什么可买的时候就退出 4.赚钱要看个股（各个板块涨跌不同） 前两条倒是好理解，第三条只要有决心就应该能做到，第四条估计我是没希望了，只能跟风了，所以只能拿1/4的资金进行操作，就当培养下知识吧，也许最终的结果是放弃，因为太费神，而且风险稍大，如果前两条收益已经不错，那么就没必要非要折腾第四条了，未来一年的需要则是，继续了解下如何研读个股，不过这些东西应该不是我的主业。

一年的总结不得不说的就是工作，我的工作很平常，没有什么大的起落，个人能力也算中上，在这个不算优秀的环境里，倒还凑合着能过，但是我想不能再这样下去了，需要稍加努力点，那么在未来的一年，应该在项目规划和代码研读上加点劲，具体就是UML的熟悉、wordpress、drupal、gallery、数据库，如果这些全能搞定，那么2010年则是一个很大成长的一年了 ，另一方面就是人缘问题了，应该需要注重一下与人沟通了。

另外不得不说的就是家庭了，总体还算不错，儿子已经一岁了，挺健康可爱，妻子也有工作，还算可以，老爸老妈过的生活也还不错，弟弟妹妹也还可以，唯一的小问题就是婆媳关系，不过相比来说已经算是不错的了，这要感激我的妻子，只是我对他们的态度问题一直没有改善，我自己也说不清，其实是很重视他们的，不过确实没有多少共同语言，而又不愿过多的去改变自己，那么在此方面未来的一年则是很简单的要求，多说话。

最后一个就是事业了，在今年将自己的事业归结为 网站+股票，今天发现这些目前还不能养活我，所以还需要在工作上加把劲的，不过工作闲暇时间还是需要努力的构建自己的事业，网站方面明年不需要开新站，仅仅把旧站维护好，升级好就行了，争取做到10年盈利2000元吧，那么必须要有3600的收入，每月就需要300的收入，估计有些难度。不过只要10年能实现这个目标，共产党不要骚扰我的话，11年就会有个很好的收获了。顺序为：剧情->blog->av->漫画->桌面，这些牵涉太多的图片了，以后再不要搞图片站了，接下来可以搞摄影知识、股票知识，之后就是发展了。
