再过20个月就要30岁了

今天突发异想，在40岁的时候要储蓄1M的money，那么在30岁的时候应该得有100k吧，随粗算下，大概还有不到20个月自己都30岁了，也就是2011年5月18日，看起来任务很艰巨，就按目前薪水的最大储蓄额每月3000，20个月也就60k，目前自有14k，外帐10k（难说那天能收回），缺口20k以上，能补缺口的只有投资收益，那必须年投资收益超过30%才有希望，另外就是额外收入，不过看起来在20个月内是没有多大希望，任务十分艰巨呀。唯一可能比较确定的是用老婆的公积金还房贷，这样可以为我节约大概5k左右，不知道两年的年底奖金能否发放6k否，任务还是比较艰巨。
接下来的生活开资应该不多了，20个月内不需要买什么其他东西了，顶多加2件春秋装，夏装可冬装已经足够，日常生活预算足够，只要小bb没啥大消费。
就看中国经济发展了，能否给我30%的投资收益了，阿门。

首先说明，我不是什么守财之人，只不过也无事，就做个小小的计划吧，总算是个目标吧。
至于40岁的1M，只能算是个希望了，必须由投资收益来提供了。年投资收益在26%，才能在100k的基础上实现。

转自：http://hi.baidu.com/bbz_dd_a/blog/item/1c962bf4b69666dff2d38588.html

公司的oracle狂慢，估计是配置没搞好吧，自己也不会只能凑合用了，google发现如下好文章，收藏之

Oracle 表删除大量数据后，即使表中只有几行记录，但用select count(*) from table 来查询发觉都不会马上出来，原因是该表的空间大了，查询起来很慢。解决的方法是把该表所占用的表空间缩小，或者说释放表空间。

alter table XXXX move; 这样处理后就释放了表空间了。但是释放表空间后，表的行号rowid会发生变化，而基于rowid的索引则会变成无效。因此该操作后必须重建索引。否则会 提示“ORA-01502: 索引'SMP.ITEMLOG_MID_IDX'或这类索引的分区处于不可用状态” 而重建索引的方法当然可以先drop掉再create ，但是这样太麻烦了，用alter index XXX rebuild 这样最快了，不会改变原来的索引结构。

体验移动g3和华为et128

在网上看到一个比较便宜的G3套餐，平均也就每月60快，跟家里宽带差不多，所以就买了，本想代替宽带使用，也就是上上网而已，在线视频基本没奢望，就是这样的需求，看来也难满足了，打开网页速度较慢，信号在90%以上，下载还算凑合，能达到40KB，总计来讲十分不满意，本来打算买联通呢，不过联通一直不争气，市面上很少，所以无奈选择了TD，看来TD还是需要长足发展的，移动已经在布局LET了（4G），应该就是明白了TD是靠不住的，所以需要更进一步，哎，既然买来了，就凑合着用吧，能满足炒股的需求。而且确实比2G是快多了。

关于华为的ET128，其自带一个小优盘功能，驱动都在里面，插上后会自动安装驱动，在我机器上出现个问题：提示安装驱动了，我也点确定了，进度条也走完了，可是实际上根本没装，所以宽带连接软件G3客户端总是提示“无设备或设备不可用”，无论我重启，还是手工安装驱动，都不行，无解，只好死马当活马医，创建一个新用户，在新用户里面进行驱动安装，居然安装成功了。设备管理器也多了华为的调制解调器，再切换会老账号，发现新硬件，可以用了。哎，看来换用户这个大法还是真有用，但是还是浪费了我半个小时时间，看来操作系统的问题，真是很莫名其妙的，有些奇怪的问题，确实很难解释，所以才会很多人经验性的重装系统。呵呵。无语。

不仅xp如此，现在的塞班更是如此，操作系统要想提高一个档次，任重而道远呀，或者说，这种问题根本是人力所不能及的，因为他确实太复杂了，除非出现智能，能自动修复，那时候，也许就是人类灭亡之时了。。

情绪与处理方式,时间管理

近来，性情稍微平缓了些，对于大多事情都不会过于干戈，不是本质有影响的事情，基本也就不去理睬了。可是本性其实没变，如果稍动下肝火，还是有点过旺，哎，看来还是不动为好，反正也没什么大不了的事情。只是这样看起来，我确实冷漠多了，那没法，谁让咱有品呢。也许达到更高的境界，需要更多的努力，那就继续吧。

学问这东西还是有些神奇的，不同层次的差别确实极大，看来还是需要进一步提高，但是却总是把时间花费在娱乐上面（也就是被暴雪给搞的，如果出了暗黑3和星际2，还让人活不让了），需要定一个死目标了。就如储蓄由结余=收入-支出 转变为 支出=收入-结余一样（也是股票给我折腾的，也算是意外收获吧），时间改为 娱乐=空闲-看书1小时，总体时间分配为工作时间处理工作的事情，业余时间看书+娱乐+锻炼。必须坚持的两点，工作时间要充分利用，看书1小时不能压缩。
