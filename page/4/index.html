高龄补贴项目

刚看到一个报道，说中国制造，就是量多质差，不用辩解，我们制作的软件，也是这样，幸好我们有一个愿意不断改进的团队。
关于用户体验，我们一直在努力，甚至都有钻牛角的态度了，相信只要有足够的资源（人才储备、时间）就能够打造一款不错的产品。
不过，目前来看，还是有些欠缺，主要是在知识量，实际上我们能够掌控的资源，也就是编码技术稍好，其他各个层面都还薄弱，人员激励、测试、设计等等。还好，我们现在所处的生产环境，还不需要我们在其他方面做的足够好，只要保证能用，稍微易用即可出炉。在没有吃饱的情况下，吃好就要推延。
目前我们要做的：
1.整合技术积累：形成一套基础程序
2.流水化生产线：需求[一直是本糊涂帐]、设计[一直都做不好]、DEMO[可用尝试]、编码[可以尝试]、测试[可以尝试]
3.提高个人技能：至少达到普通高速编码的水准
以前的成果：
1.技术积累
2.用户体验意识

高领补贴，就是这样一个演练，在数据直报我们耗费了大量的时间，4、5、6、7、8，将近5个月的时间，才基本完工。
高龄补贴采用快速迭代的方法试试效果。
1.先做基本版本，不过多考虑易用性，只考虑功能性，不考虑特殊需求，只考虑公共需求，投石问路，钓出客户个性化需求
2.客户试用，提出改进意见，进行项目优化
3.总结，整合

这样做也有坏处，如果不能很好的把握客户，第一次体验一个不够先进的版本，容易丢单，目前来看，这个方法还是更适合公司现在的状况，快速构造几个产品，快速占领空白市场，但是，为了维护客户利益，后期的优化，还是必须快速跟进。

投票器

原来是伪造投票，现在要防止伪造投票，总结下：

1.IP限制，看来只要使用 REMOTE_ADDR 基本能防止IP伪造［原理上还是可以伪造，伪造IP包发送出去，返回数据就不能返回到本机了］，针对这个可以作2次请求，第一次http请求并不记录投票结果，返回一个加密唯一数据包，让js再次提交，之后再次验证IP，应该说，万无一失了
2.验证码限制，这个只要作高端的验证码生成器，绝无问题，国外网站的一些验证码，人都看不懂，崩溃
3.设置一些cookie、header头之类的，感觉没必要，太好伪装了

最终能够防止的，也就是IP限制票数，其他的可以使用程序加人工（识别验证码验证问题等）实现快速投票 。
谁还有什么高招？哈哈，黑他服务器，黑他数据库，自己赶紧人工投几票，然后拒绝服务攻击，貌似法还不少类。

一些HTTP协议中的相关知识
获取用户IP地址的三个属性的区别(HTTP_X_FORWARDED_FOR,HTTP_VIA,REMOTE_ADDR) 分五种情况：
一、没有使用代理服务器的情况：

REMOTE_ADDR = 您的 IP
HTTP_VIA = 没数值或不显示
HTTP_X_FORWARDED_FOR = 没数值或不显示

二、使用透明代理服务器的情况：Transparent Proxies

REMOTE_ADDR = 最后一个代理服务器 IP
HTTP_VIA = 代理服务器 IP
HTTP_X_FORWARDED_FOR = 您的真实 IP ，经过多个代理服务器时，这个值类似如下：203.98.182.163, 203.98.182.163, 203.129.72.215。

这类代理服务器还是将您的信息转发给您的访问对象，无法达到隐藏真实身份的目的。

三、使用普通匿名代理服务器的情况：Anonymous Proxies

REMOTE_ADDR = 最后一个代理服务器 IP
HTTP_VIA = 代理服务器 IP
HTTP_X_FORWARDED_FOR = 代理服务器 IP ，经过多个代理服务器时，这个值类似如下：203.98.182.163, 203.98.182.163, 203.129.72.215。

隐藏了您的真实IP，但是向访问对象透露了您是使用代理服务器访问他们的。

四、使用欺骗性代理服务器的情况：Distorting Proxies

REMOTE_ADDR = 代理服务器 IP
HTTP_VIA = 代理服务器 IP
HTTP_X_FORWARDED_FOR = 随机的 IP ，经过多个代理服务器时，这个值类似如下：203.98.182.163, 203.98.182.163, 203.129.72.215。

告诉了访问对象您使用了代理服务器，但编造了一个虚假的随机IP代替您的真实IP欺骗它。

五、使用高匿名代理服务器的情况：High Anonymity Proxies (Elite proxies)

REMOTE_ADDR = 代理服务器 IP
HTTP_VIA = 没数值或不显示
HTTP_X_FORWARDED_FOR = 没数值或不显示 ，经过多个代理服务器时，这个值类似如下：203.98.182.163, 203.98.182.163, 203.129.72.215。

完全用代理服务器的信息替代了您的所有信息，就象您就是完全使用那台代理服务器直接访问对象。
