工作饱和度

现状及分析

上次领导提到，员工工作饱和度的问题。一直对此问题很不屑，这次发现，并不那么简单。一直认为“积极性”才是决定工作效率的关键，所以倾向于打造舒适、有诱惑的工作环境来提高效率。公司做的还不够，不过，效果也有待商榷。

《晓说》里面的一句话，让我深思，“没有绝对，没有黑白，只有无尽的妥协”，所以，工作饱和度也要区分人、区分环境，过大的压力导致质量问题，得不偿失。

  1. 没有良好的技术基础作为支撑
  2. 过度的工作量，引起消化不良，恶性循环

要素：

  1. 非脑力劳动，工作流程简单，安排的越多，干的越快，效率越高。
  2. 脑力劳动，例如：软件开发，此类问题，就不一定是工作压力越大，写的代码越多，越好，而是根据团队的定位，综合考量，追求质量与数量的临界点。
  3. 超越型，有个人价值观，能够按照自我安排，超越工作时间完成工作，可遇不可求。
  4. 积极型，能够在工作时间饱和工作，就现在的工作环境，也是可遇不可求。
  5. 普通型，随波逐流，能做就做，有的只是空洞的想法，没有执行力或个人能力，通常都是这个类型。

不同的人，不同的工作环境，安排不同的工作方式。
目前我们属于 脑力普通型

未来计划要做的事情

1.工作安排与指导

花费更多的时间，进行工作指导，和工作检查。

2.花费更多的时间，理顺项目脉路

曾经尝试，总是因为过度的工作内容导致，无法进展。

3.重新认定员工等级

质量对应技能水平，数量对应贡献大小。

数量：

  1. 工作内容[Svn提交的代码、Bugfree解决的问题]（生成每日、每周、每月、每季度、每年的报告，其他内容，通过bugfree填写，并指给自己完成，来处理）
  2. 对于他人的帮助（影响自己的进度）

质量：

  1. 提出的个人建议，工作、项目（目前几乎没有，暂不考虑）
  2. 工具熟练程度[快捷键的使用量25%|50%|75%]
  3. Bug数量及级别[粗心级、其他]
  4. 表述及文档能力（高级）

各取所长_成长

雾霾 平
如何写好一个文档，不是一件简单的事情，是一个综合素质的体现：文字、逻辑、心理、人文各个学科都有涉及，但好像大家对此并不十分看重，大抵认为就是一个文档而已，说明说明系统功能，就over了，对做这些没有多大兴致，这就很难办。我还是偏向，兴趣驱动的，工作中得不到乐趣，就很难将工作做得精彩。
1.天生对某件事有兴趣，比如我一直对计算机很有兴趣，现在对管理有兴趣，就很好办，给他做喜欢的事，适当指引就好了
2.没有兴趣点的，如何去做？目前只能循规蹈矩，让其按部就班，但我也不知道什么是部什么是班。
各取所长，适当的调整工作方法及内容。
很多时候，有人能够看到一些东西，有人看不到，不是能力的问题，仅仅是经历的不同，就如父辈总是教育我们好好学习，而我们不为所动一样，能改进的仅仅是还原历史的真相，讲述：经历了什么，得到了什么，至于是否能听懂，随人吧。
人力资源是个大学问，了解不多，只能现学现用了，别人的总结：性格、智力、知识，大抵就是这3个方面。

忍者

不在沉默中爆发，就在沉默中死亡，貌似忘记了，爆发中更可能死亡，只不过一个是慢性，一个是急性，好一点的急性的如果用药得当，还是有治愈的机会的，所以公司选择了爆发，结果，每三天我们就会收到一份所谓的规范，诸如：考核、工作流程等，而最近收到的是着装，看来公司真是费了很大功夫呀。

首先，必须承认，公司确实有很多人，拿着别人给的钱，还不好好被人曰，哎，悲剧，工作也没搞好，个人也没成长，我就是很典型的一个，确实，我很不成材，现在的我已经认了，如果你认为，我很菜，那么请让你自己牛逼起来。

如果说一个公司的成长，是靠每个员工辛勤工作获得的，那是真理，如果每个员工，不用管理，就能自己好好工作，那么这些员工都可以去创业了，而不用辛苦的被人剥削，而为何那么多人不去创业，且创业成功率很低，都是因为个人不因素所致，并非你想认真就能认真起来，你想执着就执着起来，你想勤奋就就勤奋起来地，所以放眼望去，尽是碌碌之辈，不用自责，我们只拿我们该得到的那份，我们心安理得即可，最大不了的，我们眼红着，自慰着，可是，现在，我们却要忍者。

如果说绝大多数员工不能良好的自慰，那么就应该有高明的领导，领导着大家，找到良好的方法和合适的频率，在愉悦中找到每个人所需要的(想领导？不是随便是个人都能做好的，先掂量掂量吧)，反之，即伤身又得不到感觉，现在公司的状况，貌似就是这样，一味的惩罚，只能让那些混的，想更多方法去混，有门路的去投明主，如我这种鸡肋，只能做个忍者。

有一些头痛了

由于近来工作压力太大，工作时间太长，感觉头有些痛了，没有明显的疼痛，只是有些微微的不适，感觉很不爽。

新来的项目负责人，很能催促工作，几乎每天都来崔几遍，加上旧工作完成的确实不是很完善，新工作又多，导致压力很大，虽然偶以“不行就不干了”的心态来调整，以求减轻压力，但还是压制的我连续几天晚上都能梦到白天在干啥，甚至在梦中找到过一个解决办法，感觉实在不爽，这不是我要的生活。而且我的身体特性都不适宜所谓的加班，按我自己的感觉，在工作上我应该是属于爆发型的，有工作可以很认真，注意力可以高度集中，但是相应的不可能持续太长时间，所以加班对我来说就是煎熬，这在和刘敏在一起的时候已经得到过验证。

本来打算在这个公司结束我的职业生涯的，看来有些难了，今年的加班是不可避免的要持续半年以上，工作压力肯定是不言而喻，项目管理也肯定不会有太大的改善，这对我来说确实是一种煎熬，没有一点乐趣。

现在只能作的一件事就是坚持坚持，希望我能活过30岁，也许好日子会来的。

09年的回顾总结(跟风一下)

09年最大的收获应该是了解了一个新领域—股市，虽然之前就有进入，但是那时候的了解，跟现在不可同日而语，这完全归功于我的一位同学、好友梁朝阳，呵呵，从他给推荐的几个blog，我了解了价值投资、趋势投资等内容，对于股市也有了新的认识，总体接受了这个领域，09年也仅仅是肤浅的了解了一些概念，根据他人的经验收录了一批股票，个人收获总结如下：
1.要买有价值的股票
2.要审时度势，把握大势
3.没事不要贪心（目前还没做到），没什么可买的时候就退出
4.赚钱要看个股（各个板块涨跌不同）
前两条倒是好理解，第三条只要有决心就应该能做到，第四条估计我是没希望了，只能跟风了，所以只能拿1/4的资金进行操作，就当培养下知识吧，也许最终的结果是放弃，因为太费神，而且风险稍大，如果前两条收益已经不错，那么就没必要非要折腾第四条了，未来一年的需要则是，继续了解下如何研读个股，不过这些东西应该不是我的主业。

一年的总结不得不说的就是工作，我的工作很平常，没有什么大的起落，个人能力也算中上，在这个不算优秀的环境里，倒还凑合着能过，但是我想不能再这样下去了，需要稍加努力点，那么在未来的一年，应该在项目规划和代码研读上加点劲，具体就是UML的熟悉、wordpress、drupal、gallery、数据库，如果这些全能搞定，那么2010年则是一个很大成长的一年了 ，另一方面就是人缘问题了，应该需要注重一下与人沟通了。

另外不得不说的就是家庭了，总体还算不错，儿子已经一岁了，挺健康可爱，妻子也有工作，还算可以，老爸老妈过的生活也还不错，弟弟妹妹也还可以，唯一的小问题就是婆媳关系，不过相比来说已经算是不错的了，这要感激我的妻子，只是我对他们的态度问题一直没有改善，我自己也说不清，其实是很重视他们的，不过确实没有多少共同语言，而又不愿过多的去改变自己，那么在此方面未来的一年则是很简单的要求，多说话。

最后一个就是事业了，在今年将自己的事业归结为 网站+股票，今天发现这些目前还不能养活我，所以还需要在工作上加把劲的，不过工作闲暇时间还是需要努力的构建自己的事业，网站方面明年不需要开新站，仅仅把旧站维护好，升级好就行了，争取做到10年盈利2000元吧，那么必须要有3600的收入，每月就需要300的收入，估计有些难度。不过只要10年能实现这个目标，共产党不要骚扰我的话，11年就会有个很好的收获了。顺序为：剧情->blog->av->漫画->桌面，这些牵涉太多的图片了，以后再不要搞图片站了，接下来可以搞摄影知识、股票知识，之后就是发展了。
