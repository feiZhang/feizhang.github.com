项目总结（统一认证平台）

最近作的项目主要有2个地方客户不甚满意。

  1. 没有提交申请磁盘阵列技术支持。
  2. 消息中心，业务系统未登录，页面转向没有处理好。

第一个问题，主要问题是：沟通不够准确。

解决方案：

  1. 成立项目组成员列表，电话、邮箱、联系人。
  2. 制作项目前期准备事项列表，发送给客户及相关人员。
  3. 鉴于客户可能不经常查看邮件，重要的内容应提供纸制文件。
  4. 制作一份项目进展情况列表，每周、每阶段发送给项目组成员。

第二个问题，主要问题是：一时疏忽，原本已经考虑到这个问题，并做出了应对方案，但没有完美部署。

解决方案：

  1. 对于用户体验的内容，尤其是关键点，增加重视度。
  2. 增加测试的工作内容，尽量还原客户现场环境。

总结：

原本想把这个项目做到所能做到的完美，还是出现了这些不必要的纰漏，遗憾并不爽，第二个问题，确实是个人的问题，应极力避免。目前的状况并未达到我个人的极限，依然出现纰漏，究其原因，还是在思想上，对于完美的追求不够深刻，以后应该继续努力。

工作中，也能完全投入，一旦闲下来，就没有了再进入的欲望。对生活提不起兴致，才是根结。

向瘦了一圈的老罗学习。

2013-07-16 工作饱和度

现状及分析

上次领导提到，员工工作饱和度的问题。一直对此问题很不屑，这次发现，并不那么简单。一直认为“积极性”才是决定工作效率的关键，所以倾向于打造舒适、有诱惑的工作环境来提高效率。虽然公司做的还不够，但是足够说明一些问题了：环境决定方法，没有一招通吃的可能。

《晓说》里面的一句话，让我深思，“没有绝对，没有黑白，只有无尽的妥协”，所以，工作饱和度也要区分人、区分环境，过大的压力导致质量问题，得不偿失。

  1. 没有良好的技术基础作为支撑
  2. 过度的工作量，引起消化不良，恶性循环

要素：

  1. 非脑力劳动，工作流程简单，安排的越多，干的越快，效率越高。
  2. 脑力劳动，例如：软件开发，此类问题，就不一定是工作压力越大，写的代码越多，越好，而是根据团队的定位，综合考量，追求质量与数量的临界点。
  3. 超越型，有个人价值观，能够按照自我安排，超越工作时间完成工作，可遇不可求。
  4. 积极型，能够在工作时间饱和工作，就现在的工作环境，也是可遇不可求。
  5. 普通型，随波逐流，能做就做，有的只是空洞的想法，没有执行力或个人能力，通常都是这个类型。

不同的人，不同的工作环境，安排不同的工作方式。 目前我们属于 脑力普通型

未来计划要做的事情

1.工作安排与指导

花费更多的时间，进行工作指导，和工作检查。

2.花费更多的时间，理顺项目脉路

曾经尝试，总是因为过度的工作内容导致，无法进展。

3.重新认定员工等级

质量对应技能水平，数量对应贡献大小。

数量：

  1. 工作内容[Svn提交的代码、Bugfree解决的问题]（生成每日、每周、每月、每季度、每年的报告，其他内容，通过bugfree填写，并指给自己完成，来处理）
  2. 对于他人的帮助（影响自己的进度）

质量：

  1. 提出的个人建议，工作、项目（目前几乎没有，暂不考虑）
  2. 工具熟练程度[快捷键的使用量25%|50%|75%]
  3. Bug数量及级别[粗心级、其他]
  4. 表述及文档能力（高级）

2013-07-11 上进心

“每个人都有上进心”，我认为这句话等同于“每个中国人都有一个武侠梦”。关键不在于你想怎么样，而在于你怎样做。

我眼中的上进

  1. 大部分的时间用于做，持续不断
  2. 在做的间歇要后头看看，向前想想，对旧事进行总结，对新事稍作计划
  3. 一颗向往美好的心，而不是被俗事所累，一旦被俗事所累，将浪费极大的精力
  4. 懂得放弃，知道自己真正想要的，不专注，一定是不求上进
  5. 阅读经典，虽然马云说：“不读书，照样成才”，但大多数时候，我们需要用书来达成顿悟，不求多，但求精
  6. 粗略了解本行业的高端技术点，并做出选择
  7. 淡定

目前的我

  1. 被琐事缠身，心绪不宁，疲于应酬。调整心绪，抽出时间，做认为正确的事情。培训、版本控制、技术升级；项目管理、交流；
  2. 好环境促进步，坏环境，也不是就一无是处。历代太祖都是逆境中成长的
  3. 保持一个好身体
  4. 对自己再狠一点，少看电影、少浏览网页，节约时间

2013-03-18 自改进的团队--不放过一个细节

1.一个销售2年的产品，仍有n多低级错误？

是研发的问题？客户都没有骚扰市场？维护部就能忍受无尽的骚扰？老板就视而不见？综合部就没有一点感觉？

不懂技术？怕得罪人？不是本岗位工作？品质观念！客户意识！有没有？

都说市场部有客户意识？给客户一个自己都没有高要求的产品，算不算有客户意识？

应对：

布道：高品质、团队观念、做事做人

实施：领导带头、相互提醒，打造公司对事不对人的风格，奖优。

2.测试平台就应该有严重问题吗？

严格要求，逐步提高，有序进步。每次忍一点就是0.99，

布道：工作内容不同，但地位相同。

实施：建立对上对下都敢言的氛围，才能构造自我改进的机制。

3.账号问题

你是一个优秀的人吗？

布道：浪费别人时间，就是犯罪。

实施：全体鄙视他。

2010-09-19

忍者

不在沉默中爆发，就在沉默中死亡，貌似忘记了，爆发中更可能死亡，只不过一个是慢性，一个是急性，好一点的急性的如果用药得当，还是有治愈的机会的，所以公司选择了爆发，结果，每三天我们就会收到一份所谓的规范，诸如：考核、工作流程等，而最近收到的是着装，看来公司真是费了很大功夫呀。

首先，必须承认，公司确实有很多人，拿着别人给的钱，还不好好被人曰，哎，悲剧，工作也没搞好，个人也没成长，我就是很典型的一个，确实，我很不成材，现在的我已经认了，如果你认为，我很菜，那么请让你自己牛逼起来。

如果说一个公司的成长，是靠每个员工辛勤工作获得的，那是真理，如果每个员工，不用管理，就能自己好好工作，那么这些员工都可以去创业了，而不用辛苦的被人剥削，而为何那么多人不去创业，且创业成功率很低，都是因为个人不因素所致，并非你想认真就能认真起来，你想执着就执着起来，你想勤奋就就勤奋起来地，所以放眼望去，尽是碌碌之辈，不用自责，我们只拿我们该得到的那份，我们心安理得即可，最大不了的，我们眼红着，自慰着，可是，现在，我们却要忍者。

如果说绝大多数员工不能良好的自慰，那么就应该有高明的领导，领导着大家，找到良好的方法和合适的频率，在愉悦中找到每个人所需要的(想领导？不是随便是个人都能做好的，先掂量掂量吧)，反之，即伤身又得不到感觉，现在公司的状况，貌似就是这样，一味的惩罚，只能让那些混的，想更多方法去混，有门路的去投明主，如我这种鸡肋，只能做个忍者。

无尽痛苦的工作

最痛苦的工作：

  1. 无聊的工作内容
  2. 没有实际意义的工作目标
  3. 质量不高的员工，包括自己
  4. 无能的管理者 我的工作环境就是这样，有的只是无尽的加班，跟吃饭没有差别了。 还有的就是无尽的bug，按照目前的状态，看来是绝无可能清除完的，大部分都不可能，如此的代码没有bug都不正常。 剩下的就是我无尽的郁闷，又没本事闪人，只能在这装洋蒜了。

难道我就是那种我最讨厌的爱抱怨的人？

痛恨的不是工作，而是我不能选择工作，没有那个本事，真菜。

2010-03-16

有一些头痛了

由于近来工作压力太大，工作时间太长，感觉头有些痛了，没有明显的疼痛，只是有些微微的不适，感觉很不爽。

新来的项目负责人，很能催促工作，几乎每天都来崔几遍，加上旧工作完成的确实不是很完善，新工作又多，导致压力很大，虽然偶以"不行就不干了"的心态来调整，以求减轻压力，但还是压制的我连续几天晚上都能梦到白天在干啥，甚至在梦中找到过一个解决办法，感觉实在不爽，这不是我要的生活。而且我的身体特性都不适宜所谓的加班，按我自己的感觉，在工作上我应该是属于爆发型的，有工作可以很认真，注意力可以高度集中，但是相应的不可能持续太长时间，所以加班对我来说就是煎熬，这在和刘敏在一起的时候已经得到过验证。

本来打算在这个公司结束我的职业生涯的，看来有些难了，今年的加班是不可避免的要持续半年以上，工作压力肯定是不言而喻，项目管理也肯定不会有太大的改善，这对我来说确实是一种煎熬，没有一点乐趣。

现在只能作的一件事就是坚持坚持，希望我能活过30岁，也许好日子会来的。
