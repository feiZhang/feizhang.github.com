2010-03-05

关于工作关于个性

中午吃饭，还给同事说，在这样的公司工作没必要着急上火的，本来就是一个混乱的体系，一个人再认真也无济于事，还影响身心健康，回来就遇到一个工作，由于自己太认真，还差点搞得跟同事不愉快，幸好现在的自己相对圆通了一些，立马道歉，挽回一些形象，毕竟都是30岁的人了，但是还是反应出我这人过于急躁的性格，好的是自己现在可以勇于面对，并纠正了。而且让我坚信，像我这样的员工，还是很有责任心的，假如能够驾驭良好，绝对战斗力强悍，但是，目前来说还真没遇到所谓的伯乐。哈哈，其实我承认我还差得远，还有很多毛病。（假如那天我有了一份急切成功的心，可以试着自己组建一个团队，试试自己的量）

事情的原因是因为路由设定的问题，由于我这几天自己在假设翻墙vpn，所以对路由设置还有些心得，刚好需要公司网管，给我们做一个到移动内网的ip路由，他死活搞不懂状况，讲半天他也弄不懂，虽有些火了，也可能是我道行太浅，需要更深理解路由问题，不过我还是认为我想的没错，一个机器想要使用那个网卡（物理的或虚拟的）送出对某个ip的访问，是可以做到的。。

2009-12-31

09年的回顾总结(跟风一下)

09年最大的收获应该是了解了一个新领域--股市，虽然之前就有进入，但是那时候的了解，跟现在不可同日而语，这完全归功于我的一位同学、好友梁朝阳，呵呵，从他给推荐的几个blog，我了解了价值投资、趋势投资等内容，对于股市也有了新的认识，总体接受了这个领域，09年也仅仅是肤浅的了解了一些概念，根据他人的经验收录了一批股票，个人收获总结如下： 1.要买有价值的股票 2.要审时度势，把握大势 3.没事不要贪心（目前还没做到），没什么可买的时候就退出 4.赚钱要看个股（各个板块涨跌不同） 前两条倒是好理解，第三条只要有决心就应该能做到，第四条估计我是没希望了，只能跟风了，所以只能拿1/4的资金进行操作，就当培养下知识吧，也许最终的结果是放弃，因为太费神，而且风险稍大，如果前两条收益已经不错，那么就没必要非要折腾第四条了，未来一年的需要则是，继续了解下如何研读个股，不过这些东西应该不是我的主业。

一年的总结不得不说的就是工作，我的工作很平常，没有什么大的起落，个人能力也算中上，在这个不算优秀的环境里，倒还凑合着能过，但是我想不能再这样下去了，需要稍加努力点，那么在未来的一年，应该在项目规划和代码研读上加点劲，具体就是UML的熟悉、wordpress、drupal、gallery、数据库，如果这些全能搞定，那么2010年则是一个很大成长的一年了 ，另一方面就是人缘问题了，应该需要注重一下与人沟通了。

另外不得不说的就是家庭了，总体还算不错，儿子已经一岁了，挺健康可爱，妻子也有工作，还算可以，老爸老妈过的生活也还不错，弟弟妹妹也还可以，唯一的小问题就是婆媳关系，不过相比来说已经算是不错的了，这要感激我的妻子，只是我对他们的态度问题一直没有改善，我自己也说不清，其实是很重视他们的，不过确实没有多少共同语言，而又不愿过多的去改变自己，那么在此方面未来的一年则是很简单的要求，多说话。

最后一个就是事业了，在今年将自己的事业归结为 网站+股票，今天发现这些目前还不能养活我，所以还需要在工作上加把劲的，不过工作闲暇时间还是需要努力的构建自己的事业，网站方面明年不需要开新站，仅仅把旧站维护好，升级好就行了，争取做到10年盈利2000元吧，那么必须要有3600的收入，每月就需要300的收入，估计有些难度。不过只要10年能实现这个目标，共产党不要骚扰我的话，11年就会有个很好的收获了。顺序为：剧情->blog->av->漫画->桌面，这些牵涉太多的图片了，以后再不要搞图片站了，接下来可以搞摄影知识、股票知识，之后就是发展了。

2009-04-25

认认真真做事,踏踏实实做人

有很多事情真是难说清，相互交叉，蛮复杂的，但是还是一管的原则，复杂的事情简单化，现在我该怎么做？ 本来是很尽心很努力地在做一件事，但是周围环境缺不尽人意，不过也正常，人生不如意十有八九，但是有些问题是不能由我处理的，而且我也处理不了，所以这件事也许就该拖沓的进行了。 每个人好像都有自己的理由，但是理由多了，问题也就多了，那么是不是每个人少一些理由，问题就会少了，事情也就顺利进行了。 青春慢慢逝去，偶也要慢慢成长。

坚持一个理念：无论如何，认认真真做事，踏踏实实做人。

2009-04-08

无聊无趣的工作

貌似现在的工作时间比以前多的多了，但是却发现更无趣了，而且好像成长更少... 目前的阶段都不清楚自己该往哪方面努力了，慢慢的工作性质稍微变化，是很少写代码了，可是也没发现有其他方面的长进，管理、构架、设计，都沾不上边，貌似就是个勤杂工，每天的工作就是将别人汇报的bug审阅，分发给相关人员，技术上也没了长进，而且所谓的管理也完全没有什么权利，说白了就是一个工作转发而已，试着寻求一下突破点。 1.带团队的技巧

  1. 主要工作列举并公告，将工作内容与人员相关联（每个人都统筹的了解进度及其所处的位置）
  2. l 2.技术构建的学习 a.了解下ibm的技术构架，不过不亲手去试试，也没有个良好的体会。

目前的问题：

  1. 系统管理混乱，具体功能点不够开放，有问题不好定位，比如：采集的问题，比对的业务逻辑，这些应该有统一的业务逻辑管理，方便所有团队人员了解
  2. bug的处理流程，提交->处理->测试->归档总结
  3. 员工没的主动性有点差
  4. 员工处理问题总是不够全面，有很多遗留问题，不够细心
